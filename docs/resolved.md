- **#7 Findings structured-output hardening** (completed: 2026-06-14) — review passes now survive a model that returns malformed or empty JSON. **Retry on unparseable/empty output:** `parseFindingsResult`/`parseVerdictsResult` (`src/review/findings.ts`, `verify.ts`) report whether the response was a *recognizable* array, distinguishing a genuine empty `[]` ("no findings"/"no verdicts" — never retried) from unparseable output (the retry trigger); the findings-only/verdicts-only wrappers delegate to them. Exports `parseFindingsResult`/`parseVerdictsResult` + `ParsedFindings`/`ParsedVerdicts`. 16 new/updated tests (parser ok/empty/unparseable, specialist retry-recover + degrade-after-retry + responseFormat wiring, verifier retry + native-JSON, Anthropic prefill, Gemini responseMimeType).
