- **#8 False-positive verification pass** (completed: 2026-06-10) — `src/review/verify.ts`: a skeptical precision step that runs before the judge. `verifyFindings` re-checks only *low-confidence* findings (below `0.8`, overridable) in a single batched "is this ACTUALLY a bug?" call; high-confidence findings are trusted and skip it, so the pass is risk-tiered (zero uncertain findings → zero cost). Verdicts drop false positives and replace confidence on survivors, so the judge's confidence floor sheds anything demoted. A missing verdict or a failed call never silently drops a finding (kept + counted). Wired through `runReview`/`reviewPullRequest` (default on; `--no-verify` to skip) with drop/demote/unverified counts — and verifier failures — surfaced in the review notes. 15 new unit tests. Branch `false-positive` (`50bc333`…`04b0d1e`).
