- **#8 False-positive verification pass** (completed: 2026-06-10) ‚Äî `src/review/verify.ts`: a skeptical precision step that runs before the judge. `verifyFindings` re-checks only *low-confidence* findings (below `0.8`, overridable) in a single batched "is this ACTUALLY a bug?" call; high-confidence findings are trusted and skip it, so the pass is risk-tiered (zero uncertain findings ‚Üí zero cost).
